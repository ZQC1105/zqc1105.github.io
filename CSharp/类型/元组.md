## [元组类型](https://learn.microsoft.com/zh-cn/dotnet/csharp/language-reference/builtin-types/value-tuples)

在.NET中，**ValueTuple** 和 **Tuple** 都是用来表示一组相关值的轻量级数据结构，但它们的实现和使用方式有所不同。以下是它们的命名和核心区别：

---

### **1. Tuple（System.Tuple）**
- **命名**：直接称为 **"Tuple"** 或 **"引用元组"**（因为它是一个**类**，引用类型）。
- **特点**：
  - 是**引用类型**（`class`），存储在堆上。
  - 不可变（创建后无法修改成员的值）。
  - 成员通过 `Item1`, `Item2` 等属性访问。
  - 语法示例：
    ```csharp
    var tuple = new Tuple<int, string>(1, "hello");
    int id = tuple.Item1;
    string name = tuple.Item2;
    ```

---

### **2. ValueTuple（System.ValueTuple）**
- **命名**：称为 **"ValueTuple"** 或 **"值元组"**（因为它是一个**结构体**，值类型）。
- **特点**：
  - 是**值类型**（`struct`），存储在栈上，性能更高。
  - 可变（创建后可以修改成员的值）。
  - 支持**命名成员**（C# 7.0+），语义更清晰。
  - 语法示例（推荐方式）：
    ```csharp
    var person = (Id: 1, Name: "hello"); // 编译器自动生成ValueTuple
    person.Name = "world"; // 可直接修改
    ```

---

### **关键区别总结**
| **特性**       | **Tuple**               | **ValueTuple**               |
|----------------|-------------------------|------------------------------|
| **类型**       | 引用类型（`class`）      | 值类型（`struct`）            |
| **性能**       | 堆分配，有GC压力         | 栈分配，无GC压力              |
| **可变性**     | 不可变                   | 可变                          |
| **成员命名**   | 仅支持 `Item1`, `Item2`  | 支持自定义命名（如 `Id`, `Name`） |
| **语法简洁性** | 需显式 `new Tuple<...>`  | 支持简写 `(int, string)` 或命名成员 |

---

### **为什么需要ValueTuple？**
- **性能优化**：避免堆分配（如高频调用时）。
- **语义化命名**：`person.Name` 比 `person.Item2` 更易读。
- **C# 7.0+ 原生支持**：语言级集成（如解构、模式匹配）。

---

### **使用场景建议**
- **用ValueTuple**：现代C#开发中**优先选择**，尤其需要性能或命名成员时。
- **用Tuple**：仅在与旧代码（如.NET Framework 4.0/4.5）兼容时。

---

总结：**Tuple是“老一代引用元组”，ValueTuple是“新一代值元组”**，后者在性能和语法上全面占优。
