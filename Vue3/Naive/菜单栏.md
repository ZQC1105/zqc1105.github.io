这个问题是因为你的 `activeKey` 在刷新页面后会重置为 `null`，而菜单组件无法自动识别当前路由并设置激活状态。以下是几种解决方案：

## 方案1：使用计算属性自动匹配路由（推荐）

```vue
<script setup lang="ts">
import { RouterLink, RouterView, useRoute } from 'vue-router'
import type { MenuOption } from 'naive-ui'
import { h, computed } from 'vue'
import { useRoute } from 'vue-router'

const route = useRoute()

// 根据当前路由自动设置 activeKey
const activeKey = computed(() => {
  // 根据路由名称设置对应的 key
  if (route.name === 'home') return 'go-back-home'
  if (route.name === 'about') return 'go-about-page'
  return null
})

const menuOptions: MenuOption[] = [
  {
    label: () =>
      h(
        RouterLink,
        {
          to: {
            name: 'home',
            params: {
              lang: 'zh-CN'
            }
          }
        },
        { default: () => '首页' }
      ),
    key: 'go-back-home',
  },
  {
    label: () =>
      h(
        RouterLink,
        {
          to: {
            name: 'about',
            params: {
              lang: 'zh-CN'
            }
          }
        },
        { default: () => '关于' }
      ),
    key: 'go-about-page',
  },
]
</script>
```

## 方案2：使用路由路径作为 key（更通用）

```vue
<script setup lang="ts">
import { RouterLink, RouterView, useRoute } from 'vue-router'
import type { MenuOption } from 'naive-ui'
import { h, computed } from 'vue'

const route = useRoute()

// 使用当前路由路径作为 activeKey
const activeKey = computed(() => route.path)

const menuOptions: MenuOption[] = [
  {
    label: () =>
      h(
        RouterLink,
        {
          to: {
            name: 'home',
            params: {
              lang: 'zh-CN'
            }
          }
        },
        { default: () => '首页' }
      ),
    key: '/', // 对应首页路由
  },
  {
    label: () =>
      h(
        RouterLink,
        {
          to: {
            name: 'about',
            params: {
              lang: 'zh-CN'
            }
          }
        },
        { default: () => '关于' }
      ),
    key: '/about', // 对应关于页路由
  },
]
</script>
```

## 方案4：使用 NMenu 的路由集成（如果你使用的是 Naive UI）

如果你使用的是 Naive UI，可以使用它内置的路由集成功能：

```vue
<script setup lang="ts">
import { RouterLink, RouterView, useRoute } from 'vue-router'
import type { MenuOption } from 'naive-ui'
import { h } from 'vue'

const menuOptions: MenuOption[] = [
  {
    label: () =>
      h(
        RouterLink,
        {
          to: {
            name: 'home',
            params: {
              lang: 'zh-CN'
            }
          }
        },
        { default: () => '首页' }
      ),
    key: '/',
  },
  {
    label: () =>
      h(
        RouterLink,
        {
          to: {
            name: 'about',
            params: {
              lang: 'zh-CN'
            }
          }
        },
        { default: () => '关于' }
      ),
    key: '/about',
  },
]
</script>

<template>
  <div class="main-container">
    <!-- 使用 router 属性让 NMenu 自动处理路由激活状态 -->
    <n-menu
      mode="horizontal"
      :options="menuOptions"
      :router="true"
      responsive
    />
    <main class="router-view-container">
      <RouterView />
    </main>
  </div>
</template>
```

**推荐使用方案1或方案4**，它们都能确保在页面刷新后菜单项仍然保持正确的激活状态。

另外，我注意到你的模板中有个空的 `n-split` 组件，没有实际内容，可以考虑移除：

```vue
<template>
  <div class="main-container">
    <!-- 直接使用 n-menu，不需要 split -->
    <n-menu
      v-model:value="activeKey"
      mode="horizontal"
      :options="menuOptions"
      responsive
    />
    
    <main class="router-view-container">
      <RouterView />
    </main>
  </div>
</template>
```
