
一、为什么断点老不生效？
--------------------------------------------------
浏览器只认 **HTTP 路径**（localhost:5173/...）里的代码；  
你打在 **file://** 或 **Workspace** 上的断点只是“影子文件”，永远不会停。

--------------------------------------------------
二、3 步选对文件（Chrome/Edge 通用）
--------------------------------------------------
1. F12 → Sources → `Ctrl|Cmd + P`  
2. 输入 `TodoList.vue`  
3. 选 **带端口** 的那一条，例如  
   `▶ localhost:5173/src/components/TodoList.vue`  
   （没端口的一律无视）

--------------------------------------------------
三、调试快捷键对照（记住 3 个就够）
--------------------------------------------------
| 操作      | VS 习惯 | 浏览器实际键 |
| --------- | ------- | ------------ |
| 刷新/重载 | F5      | F5           |
| 继续执行  | F5      | **F8** ⭐     |
| 单步跳过  | F10     | F10          |
| 单步进入  | F11     | F11          |

--------------------------------------------------
四、完整流程 10 秒版
--------------------------------------------------
1. 在「HTTP 路径」下打红点  
2. F5 刷新（让断点注册）  
3. 页面上点按钮 → 自动暂停  
4. **F8 继续 / F10 跳过**

--------------------------------------------------
五、一眼识别“真假文件”
--------------------------------------------------
✅ 真：路径含 `localhost:5173` 或 `localhost:3000`  
❌ 假：路径以 `file://` 开头，或在 **Filesystem** 节点下

--------------------------------------------------
六、常见坑速排
--------------------------------------------------
- 装了 **Workspace** 手动映射？右键 → Remove from workspace  
- 改完代码没刷新？**必须 F5** 重新加载新 bundle  
- 用了 VS Code 插件？**也要在浏览器里再打一次断点**；插件断点 ≠ 浏览器断点 

--------------------------------------------------
七、一句话总结
--------------------------------------------------
**“HTTP 路径打断点，F5 刷新，F8 继续。”**  
记住这句，Vue+Vite 调试再不会踩坑。
