在 SQL Server 中，实现一对一关系主要有以下几种方法：

### 1. 使用唯一外键（Most Common）
- **原理**：在从表中创建一个外键列，并给该列加 UNIQUE 约束，确保每个主表记录只能对应一个从表记录。
- **示例**：
  ```sql
  -- 主表
  CREATE TABLE Users (
      UserID INT PRIMARY KEY,
      UserName NVARCHAR(50)
  );

  -- 从表
  CREATE TABLE UserProfiles (
      ProfileID INT PRIMARY KEY,
      UserID INT UNIQUE,          -- 唯一外键
      Bio NVARCHAR(200),
      FOREIGN KEY (UserID) REFERENCES Users(UserID)
  );
  ```
- **特点**：
  - 从表的外键必须唯一。
  - 主表记录可以没有对应的从表记录（可选一对一），也可以强制要求必须有（通过 NOT NULL 和触发器或应用逻辑实现）。

---

### 2. 共享主键（Shared Primary Key）
- **原理**：主表和从表使用相同的主键值，从表的主键同时作为外键引用主表。
- **示例**：
  ```sql
  -- 主表
  CREATE TABLE Users (
      UserID INT PRIMARY KEY,
      UserName NVARCHAR(50)
  );

  -- 从表
  CREATE TABLE UserProfiles (
      UserID INT PRIMARY KEY,     -- 既是主键也是外键
      Bio NVARCHAR(200),
      FOREIGN KEY (UserID) REFERENCES Users(UserID)
  );
  ```
- **特点**：
  - 主表和从表的主键值必须相同。
  - 插入从表记录前，必须先插入主表记录。
  - 更严格的一对一，因为主键必须唯一且非空。

---

### 3. 复合主键 + 唯一约束（较少用）
- **原理**：通过复合主键和唯一约束组合实现一对一，但这种方式复杂且不直观，通常不推荐。
- **示例**（不推荐）：
  ```sql
  CREATE TABLE TableA (
      A_ID INT PRIMARY KEY,
      A_Data NVARCHAR(50)
  );

  CREATE TABLE TableB (
      A_ID INT,
      B_ID INT,
      B_Data NVARCHAR(50),
      PRIMARY KEY (A_ID, B_ID),
      UNIQUE (A_ID)               -- 确保 A_ID 唯一
  );
  ```

---

### 总结对比表：

| 方法           | 是否推荐 | 特点说明                                                                 |
|----------------|----------|--------------------------------------------------------------------------|
| 唯一外键       | ✅ 推荐   | 灵活、常见，支持可选一对一，外键唯一即可                                 |
| 共享主键       | ✅ 推荐   | 严格一对一，主键值必须相同，插入顺序有要求                              |
| 复合主键+唯一  | ❌ 不推荐 | 复杂、不直观，维护困难                                                  |

---

### 最佳实践建议：
- **可选一对一**（如用户与用户详情）：使用**唯一外键**。
- **严格一对一**（如用户与身份证）：使用**共享主键**。
